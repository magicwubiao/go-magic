# Cortex Agent 架构总览

> **Cortex** 是核心认知引擎的新名称。**Cortex** 仍被支持以保持向后兼容。

## 概述

go-magic 是一个高性能、超轻量级的 Go 语言 AI Agent 实现，灵感来源于 Nous Research 的 Hermes Agent。本项目实现了完整的三层认知架构和六大自进化系统。

**核心原则**：
- 用得越多，越懂你
- 从经验中自动学习
- 记忆是资产，不是负担
- 前缀缓存优先，保护成本

## 三层认知架构

### Layer 1: 感知层 (Perception)

**目标**：理解用户说了什么

**输入**：用户消息文本 + 对话历史
**输出**：`PerceptionResult` 结构化理解结果

**核心能力**：

| 能力 | 说明 | 实现状态 |
|------|------|----------|
| 意图分类 | 7 种类型自动识别 | ✅ |
| 复杂度评估 | 3 级，决定资源配置 | ✅ |
| 实体提取 | 语言/文件/工具 自动识别 | ✅ |
| 噪声检测 | 不完整/模糊/矛盾 检测 | ✅ |
| 上下文提示 | 是否需要参考历史 | ✅ |

**代码位置**：`internal/perception/`

### Layer 2: 决策层 (Cognition)

**目标**：决定应该怎么做

**输入**：感知结果 `PerceptionResult`
**输出**：`Decision` 完整决策 + `ExecutionPlan` 执行计划

**核心能力**：

| 能力 | 说明 | 实现状态 |
|------|------|----------|
| 任务拆解 | 复杂任务 → N 个有序步骤 | ✅ |
| 依赖管理 | DAG 式依赖检测，按序执行 | ✅ |
| 动态调整 | 失败 → 自动添加重试步骤 | ✅ |
| Max Turns | Simple=8, Medium=15, Advanced=25 | ✅ |
| 子代理决策 | Advanced 自动启用子代理 | ✅ |
| 检索提示 | 按实体/上下文生成搜索关键词 | ✅ |
| 澄清检测 | 有噪声 → 生成澄清问题 | ✅ |

**代码位置**：`internal/cognition/`

### Layer 3: 执行层 (Execution)

**目标**：真正把事做好，可中断可恢复

**输入**：决策结果 `Decision` + 执行计划 `ExecutionPlan`
**输出**：执行状态 + 结果校验

**核心能力**：

| 能力 | 说明 | 实现状态 |
|------|------|----------|
| 检查点机制 | 每步结果持久化到磁盘 | ✅ |
| 断点续传 | 重启后从断点继续 | ✅ |
| 结果校验 | 预期 vs 实际 对比 | ✅ |
| 进度展示 | 用户可见的进度条 | ✅ |

**代码位置**：`internal/execution/`

## 六大自进化系统

### 系统一: 用户消息触发器

**位置**：`internal/trigger/`
**状态**：✅ 完成

每一条用户消息：
1. 轮次计数器 +1
2. 如果是新任务 → 初始化技能创建流程
3. 如果轮次 % N == 0 → 触发 Nudge 信号

### 系统二: Nudge 机制

**位置**：`internal/trigger/` + `internal/review/`
**状态**：✅ 完成

- Nudge 信号触发后，异步启动后台复盘
- **绝不阻塞**用户对话
- 可以配置阈值（默认 15 轮）

### 系统三: 后台复盘系统

**位置**：`internal/review/`
**状态**：✅ 完成

后台异步进行：
1. 分析最近 N 轮的工具调用模式
2. 识别重复出现的工作流
3. 生成技能草稿（Markdown 格式）
4. 提炼核心结论写入 `MEMORY.md`
5. 完整复盘报告持久化到磁盘

### 系统四: 双文件存储 + 冻结快照

**位置**：`internal/memory/snapshot_manager.go`
**状态**：✅ 完成

**核心创新**：

```
本轮对话进行中:
    MEMORY.md 已更新 ← 写入磁盘
    但系统提示 = 本轮开始时的冻结快照 ← 不加载变更
    ↓
    前缀缓存 HIT 率 100% ← 成本保护

下一轮对话 / 新会话:
    刷新冻结快照
    加载最新的 MEMORY.md
```

**字符限制设计**：
- `MEMORY.md`：2,200 字符上限
- `USER.md`：1,375 字符上限

> 字符上限不是技术限制，是设计选择。它倒逼 Agent 提炼精华而不是堆砌原文。

### 系统五: 全息记忆检索

**位置**：`internal/memory/fts_store.go`
**状态**：✅ 完成

- 所有会话历史存入 SQLite
- FTS5 全文索引，BM25 相关性排序
- 重要性分级 (1-10)，智能清理
- 洞见 (insight) 专门存储
- 记忆版本管理，可以回滚到任意时间点

### 系统六: 技能自进化

**位置**：`internal/skills/auto_enhanced.go`
**状态**：✅ 完成

**渐进式披露**：

| 等级 | 内容 | Token 消耗 |
|------|------|-----------|
| Level 0 | 技能名称 + 一句话描述 | ~50 |
| Level 1 | 完整使用说明 + 步骤 | ~500 |
| Level 2 | 参考文档 + 示例代码 | ~2,000+ |

根据任务复杂度自动选择披露等级，平衡能力与成本。

## 成本优化策略

### 前缀缓存保护

**传统方案**（不使用冻结快照）：
```
每轮对话系统提示都不同
  → 前缀缓存 MISS
  → 每轮都是全量 tokens 计费
  → Anthropic Claude: ~$0.20 / 轮
  → 100 轮对话: ~$20.00
```

**Cortex 方案**（使用冻结快照）：
```
多轮使用相同系统提示
  → 前缀缓存 HIT 率 ~90%
  → 缓存命中时成本仅为原来的 10%
  → Anthropic Claude: ~$0.02 / 轮
  → 100 轮对话: ~$2.00
```

**节省：90% API 成本** 💰
